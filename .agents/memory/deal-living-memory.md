# Deal sweep living memory

The deal sweep no longer overwrites its analysis. Each `deal_records.record`
jsonb carries `packets[]` (durable insight store = source of truth) and
`deltas[]` (newest-first change log), alongside `hard`/`ai`. No DB migration ‚ÄĒ
they live inside the existing record. `deal_engine_packets.py` is the pure,
deterministic core (`reconcile`, `extract_candidates`, `project_into_ai`).

## Non-obvious decisions (be consistent with these)

- **ai.* is a PROJECTION; packets are the source of truth.** After a sweep,
  packet-backed `ai.*` item lists (explicit/implicit requirements, stakeholder
  map, competitors, vulnerabilities, open_deliverables, best_practice flags) are
  regenerated by `project_into_ai` from active+dormant packets. Don't write those
  lists directly expecting them to survive ‚ÄĒ they are recomputed every sweep.
  **Why:** retain-on-absence (dropped facts must still show) requires the lists
  to come from the durable store, not the latest agent output.

- **Candidates are derived SERVER-SIDE from the agent's emitted ai.*/hard**, NOT
  from a separate LLM `candidate_packets` array (deviates from the original task
  plan on purpose). **Why:** guarantees a section the agent DID emit can never be
  wiped by projection (candidates == the agent's current items), while still
  merging retained dormant facts. The prompt only gives a "reuse the same
  wording" nudge + a known-topics list for key stability.

- **Stable key = `type:slug(subject)`.** Entity resolution across daily sweeps
  depends on the agent phrasing a recurring topic the same way. Slight rephrasing
  ‚Üí new key ‚Üí old goes dormant + new added (mild duplication, never data loss).

- **Change detection compares ONLY per-type significant fields**
  (`_SIGNIFICANT_FIELDS`), not the whole value. **Why:** a daily re-sweep that
  only refreshes a quote/date would otherwise log a `changed` delta every day and
  flood the "what changed" feed. The stored value is still refreshed to latest;
  only meaningful field moves emit a delta.

- **Retention/absence policy:** DURABLE types (requirement, pain, stakeholder,
  risk, hygiene, champion) absent from a sweep ‚Üí `dormant` (retained, still
  projected, last_confirmed unchanged). Non-durable (product_scope, competitor,
  commitment) ‚Üí left active. Nothing is ever blind-deleted; `superseded`/
  `resolved` only on explicit candidate signal (`supersedes`/`resolves`/
  `value.status==resolved`).

- **champion & product_scope are singleton packets** (subject `champion` /
  `scope`) tracked for the change feed + history; champion is NOT rendered into
  stakeholder_map but IS backfilled into `ai.champion_strength` only when the
  agent omits it (never overrides an emitted champion).

- **Baseline seeding ‚Č† change.** Deals swept BEFORE living memory have no packet
  store, so the change feed is empty for them. The one-time backfill (`seed_packets`
  ‚Üí `store.backfill_packets`, `POST /api/deal-engine/backfill-packets`, or
  `scripts/backfill_deal_packets.py`) derives packets from each record's EXISTING
  `ai.*`/`hard` and attaches them with an EMPTY delta log ‚ÄĒ seeding pre-existing
  facts must NOT emit `added` deltas (would flood the feed with weeks-old facts as
  "new"). It does NOT re-project `ai` (prior analysis preserved byte-for-byte).
  Idempotent: keys off `schema_version >= 2` (set by the seed), so records already
  carrying packets are skipped and a re-run never re-seeds or double-charges.

## Wiring & ops
- `analyze_one` loads the prior record before invoking (passes `known_topics_block`),
  then parse ‚Üí reconcile ‚Üí project ‚Üí upsert. Reconcile is wrapped in try/except so
  it NEVER fails a sweep (falls back to the agent's record). Sets `schema_version=2`.
  Deltas capped by `DEAL_DELTA_CAP` (default 200). Shared path ‚Üí covers
  bulk/manual/salesforce-trigger sweeps.
- Change feed: `GET /api/deal-engine/deltas?owner=&limit=` (book + byOwner rollup)
  and `/deltas/{opp_id}`.
- Tests: `tests/test_deal_engine_packets.py` (deterministic core; covers the 3
  acceptance cases + noise suppression + champion backfill).
