# Cost, performance & hang protection

## Cost & performance
- **Anthropic prompt caching** (`anthropic_cache.py`): `CachedChatAnthropic` auto-injects `cache_control: ephemeral` on the system prompt block and the last tool definition (caches the entire tools array). Billing: uncached input 1.00×, cache write 1.25×, cache read 0.10×, output 1.00×. `chat_usage.input_tokens` stores grand total; `[USAGE]` log line reports cache_write / cache_read / hit% per chat.
- **Tool-output summariser:** `AgentManager._wrap_mcp_tool` (`server.py:663`) summarises any single tool response > `TOOL_RESPONSE_SUMMARIZE_THRESHOLD` (default **5,000 chars**) via `gpt-4o-mini` before context append. Full raw payload still saved to `mcp_output/` and `chat_messages` for forensics/UI. Avoma tools exempt (`_AVOMA_NO_TRUNCATE_TOOLS`). Cuts per-step replay 60–80% on long ABM runs.
- **Tool-level allow/denylist (cost task #40, 2026-05-22):** finer-grained than the server-level `MCP_SERVER_ALLOWLIST`/`MCP_SERVER_DENYLIST`. Two comma-separated env vars of fnmatch globs applied in `_rebuild_flat_cache` after wrapping:
  - `MCP_TOOL_ALLOWLIST` (e.g. `"salesforce_*,lemlist_*,zi_*,apollo_*,zerobounce_*"`) — when non-empty, ONLY matching MCP tools register with the agent; denylist ignored.
  - `MCP_TOOL_DENYLIST` (e.g. `"mailchimp_*,showpad_*,linkedin_*,eloqua_*,gojiberry_*"`) — drops matching tools while leaving the servers running (still reachable via the `/mcp` endpoint).
  - Custom tools (`web_search`, `search_knowledge`, `send_to_clay`, `web_scrape`, `web_search_with_urls`, `get_current_time`, `browser_*`) are NEVER filtered.
  - Each filter pass logs `[TOOL-FILTER]` with kept/dropped counts and first 10 dropped names.
  - **Active safety denylist (Salesforce writes):** `MCP_TOOL_DENYLIST` is set to `create_record,update_record,delete_record,create_contact,update_contact_email_from_ai,set_hot_abm_status,create_task`. This enforces the hard rule that the system must **never** write to Salesforce — applied to the shared catalog, so both the main chat agent and the analysis AI cells (`analysis_engine.get_all_tools` / `set_tool_provider`) lose these tools. Salesforce **read** tools (`soql`, `get_record`, `describe_object`, `list_objects`, `search`) remain. Do not remove these entries.
- **Intra-run context-trim middleware (cost task #40, 2026-05-22):** `agent_checklist/context_trim_middleware.py` `ContextTrimMiddleware`. Hooks `awrap_model_call`, so it fires before EVERY LLM call inside a single `astream()` (not just at the entry-site pre-trim). When `_estimate_tokens(request.messages) > CONTEXT_TRIM_THRESHOLD_TOKENS` (default **60,000**), walks oldest→newest and replaces `.content` on every `ToolMessage` longer than `CONTEXT_TRIM_PLACEHOLDER_MAX_CHARS` (default 400) with a one-line placeholder `[Earlier <tool> result summarised: N chars omitted; first 200: ...]`. Never drops/reorders messages and never touches the last `CONTEXT_TRIM_KEEP_RECENT_MESSAGES` (default 10), so tool_call_id pairing always holds (Anthropic API requires matching `tool_use`/`tool_result` pairs). Toggle via `CONTEXT_TRIM_ENABLED=false`. Logs `[CONTEXT-TRIM] before → after tokens` on each trim. Targets the real cost driver on long ABM runs (chat `15dbbb61` cost $17.40, 152 tool_calls) — historical `tool_result` blocks replayed on every step.
- **Pre-astream summarizer (cross-turn):** `ContextWindowManager.summarize_conversation_history` runs once at the 5 entry points (REST `/api/chat`, async `/api/chat/async`, structured + structured async, **and WS `/ws/chat`** — parity fix 2026-05-22) when restored Supabase history exceeds `CONVERSATION_SUMMARIZE_TOKEN_THRESHOLD` (default lowered 100K→**60K** on 2026-05-22). Keeps last `CONVERSATION_KEEP_RECENT_MESSAGES` (default 20) verbatim, summarises older via `gpt-4o-mini`.
- **Verifier (advisory only):** `verifier/` package runs after every turn, evaluates the registered flow, writes a `verifier_report` row (read via `GET /api/chat/{chat_id}/verifier_report`). Auto-remediation re-prompt loop **removed 2026-05-20** (doubled cost, rarely fixed gaps). `verifier/remediation.py` retained as dead code.

## Hang protection (added 2026-05-20)
1. Anthropic SDK `max_retries` 8→2; per-call `LLM_REQUEST_TIMEOUT_S` 120→180s. Worst-case retry budget ~6 min, not 16.
2. Watchdog: every `agent.astream` chunk wrapped in `asyncio.wait_for(WATCHDOG_STALL_SECONDS=180)` across all 3 astream sites (`_agent_astream_autocontinue` + WS `/ws/chat`). On timeout: `aclose()` + `RuntimeError` → caught by existing handlers → loud `type='error'` row.
3. `search_knowledge` per-**step** cap + run-scoped dedupe; the step counter resets on each auto-continue iteration (`reset_search_knowledge_step`) so a legitimately RAG-heavy multi-step run doesn't accumulate into the runaway-loop cancel, while the dedupe memory + full counter reset still fire at start AND in `finally` of both run paths.
